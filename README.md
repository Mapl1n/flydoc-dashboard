# 📂 FlyDoc 智能档案平台 — 统一入口

三个独立微服务的反向代理 Dashboard：RAG 问答引擎 + 文档处理流水线 + 知识图谱构建器。

![Go](https://img.shields.io/badge/Go-1.23-00ADD8?logo=go)

## 🎯 一句话

一个端口管理全部三个项目。Pipeline 处理完的文档一键同步到 RAG 引擎可搜索。

## 🏗️ 架构

```
                    ┌─ /s1 → RAG 问答引擎 (:8081)
Dashboard :8484 ───├─ /s2 → 文档处理流水线 (:8082)
                    └─ /s3 → 知识图谱构建器 (:8083)
                    
              /sync → Pipeline 文档 → RAG 索引（一键同步）
```

## 🚀 启动

```bash
go run .
# 打开 http://localhost:8484
```

三个后端服务需先运行（:8081/:8082/:8083）。

## 📡 端点

| 路由 | 说明 |
|------|------|
| `/` | Dashboard 首页 |
| `/s1/` | RAG 引擎反向代理 |
| `/s2/` | 文档流水线反向代理 |
| `/s3/` | 知识图谱反向代理 |
| `/sync` | 同步 Pipeline 文档到 RAG |
